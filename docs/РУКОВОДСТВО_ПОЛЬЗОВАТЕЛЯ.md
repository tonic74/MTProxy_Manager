## Введение

### Что такое MTProxy Manager?

MTProxy Manager - это веб-панель для управления MTProto прокси-серверами Telegram. Система позволяет:

- Добавлять и управлять несколькими прокси-серверами
- Генерировать ссылки для подключения пользователей
- Ротировать секреты для обхода блокировок
- Мониторить статус и активность серверов
- Автоматизировать управление через Telegram-бота

### Для кого это руководство?

Это руководство предназначено для администраторов, которые будут использовать панель управления для обслуживания прокси-серверов и предоставления доступа пользователям.

### Требования

- Современный веб-браузер (Chrome, Firefox, Safari, Edge)
- Доступ к интернету
- Учетные данные для входа (если настроена авторизация)

## Начало работы

### Вход в панель управления

1. Откройте браузер
2. Перейдите по адресу вашей панели управления (например: `https://mtproxy.example.com`)
3. Если настроена авторизация, введите логин и пароль

### Первый запуск

При первом входе панель будет пустой. Вам нужно:

1. Убедиться, что MTProxy сервер установлен и запущен на VPS
2. Добавить информацию о сервере в панель управления
3. Сгенерировать или ввести секрет
4. Проверить подключение

## Управление прокси-серверами

### Добавление нового прокси

1. Нажмите кнопку **"+ Добавить прокси"**
2. Заполните форму:

| Поле | Описание | Пример |
|------|----------|--------|
| **Название** | Понятное имя для сервера | `Germany-1` или `DE-Frankfurt` |
| **IP-адрес сервера** | Адрес вашего VPS | `185.123.45.67` |
| **Порт** | Порт MTProxy (по умолчанию 443) | `443` |
| **Секрет** | 32-символьный hex-код или оставьте пустым для автогенерации | `ee...` |
| **Fake TLS домен** | Домен для маскировки (опционально) | `www.google.com` |

3. Нажмите **"Сохранить"**

### Типы секретов

Система поддерживает два типа секретов:

#### Обычный секрет (устаревший)
- 32 символа (hex)
- Пример: `d41d8cd98f00b204e9800998ecf8427e`
- Легко блокируется DPI

#### Fake TLS секрет (рекомендуется)
- Начинается с `ee`
- Маскирует трафик под HTTPS
- Пример: `ee1234...` + закодированный домен
- Сложнее обнаружить и заблокировать

### Редактирование прокси

1. Найдите нужный сервер в таблице
2. Нажмите кнопку **"..."** (три точки) в столбце "Действия"
3. Выберите **"Редактировать"**
4. Измените необходимые поля
5. Нажмите **"Сохранить"**

### Удаление прокси

1. Нажмите **"..."** напротив нужного сервера
2. Выберите **"Удалить"**
3. Подтвердите удаление

> **Внимание:** Удаление записи из панели не останавливает сам MTProxy сервер на VPS. Это только удаляет информацию из системы управления.

### Статусы серверов

| Статус | Индикатор | Описание |
|--------|-----------|----------|
| **Активен** | Зеленый | Сервер работает и доступен |
| **Недоступен** | Красный | Сервер не отвечает |
| **Проверка** | Желтый | Идет проверка статуса |
| **Отключен** | Серый | Сервер отключен вручную |

## Работа со ссылками

### Типы ссылок подключения

Для каждого прокси доступны несколько форматов ссылок:

#### 1. tg:// ссылка (основная)
```
tg://proxy?server=185.123.45.67&port=443&secret=ee...
```
- Открывается напрямую в Telegram
- Работает на всех платформах
- Рекомендуется для большинства пользователей

#### 2. t.me ссылка (веб)
```
https://t.me/proxy?server=185.123.45.67&port=443&secret=ee...
```
- Открывается через браузер
- Перенаправляет в приложение Telegram
- Удобна для отправки в чатах

### Копирование ссылки

1. Нажмите **"..."** напротив нужного сервера
2. Выберите **"Копировать ссылку"**
3. Выберите формат (tg:// или t.me)
4. Ссылка скопирована в буфер обмена

### Отправка ссылки пользователям

**Рекомендации:**

- Отправляйте ссылку личным сообщением, а не в публичные чаты
- Объясните пользователю, что после перехода по ссылке нужно нажать "Подключить"
- Предупредите, что при ротации секрета нужно будет переподключиться

### QR-код

Для удобства можно сгенерировать QR-код:

1. Нажмите **"..."** напротив сервера
2. Выберите **"QR-код"**
3. Покажите код пользователю или сохраните изображение

## Ротация секретов

### Зачем нужна ротация?

- **Обход блокировок** - если секрет заблокирован, новый секрет позволит продолжить работу
- **Безопасность** - периодическая смена секрета ограничивает несанкционированный доступ
- **Управление доступом** - можно "отключить" старых пользователей, сменив секрет

### Ротация одного сервера

1. Нажмите **"..."** напротив нужного сервера
2. Выберите **"Ротировать секрет"**
3. Подтвердите действие
4. Новый секрет будет сгенерирован автоматически

> **Важно:** После ротации все текущие пользователи потеряют подключение. Им нужно будет переподключиться по новой ссылке.

### Ротация всех серверов

1. Нажмите кнопку **"Ротировать все"** в верхней части страницы
2. Подтвердите массовую ротацию
3. Секреты будут обновлены на всех активных серверах

**Используйте осторожно!** Это отключит всех пользователей на всех серверах.

### Автоматическая ротация

Можно настроить автоматическую ротацию по расписанию:

1. Перейдите в **Настройки**
2. Найдите раздел **"Автоматическая ротация"**
3. Включите функцию
4. Выберите интервал:
   - Каждый день
   - Каждую неделю
   - Каждый месяц
5. Укажите время ротации

## Telegram-бот

### Возможности бота

Если настроен Telegram-бот, вы можете управлять прокси прямо из мессенджера:

| Команда | Описание |
|---------|----------|
| `/start` | Начало работы с ботом |
| `/list` | Список всех прокси |
| `/status` | Статус всех серверов |
| `/link <имя>` | Получить ссылку на конкретный прокси |
| `/rotate <имя>` | Ротировать секрет |
| `/rotateall` | Ротировать все секреты |
| `/help` | Справка по командам |

### Пример использования

```
Вы: /list

Бот: Ваши прокси-серверы:
     1. DE-1 (185.123.45.67) - Активен
     2. NL-2 (89.12.34.56) - Активен
     3. US-1 (45.67.89.12) - Недоступен

Вы: /link DE-1

Бот: Ссылка для подключения к DE-1:
     tg://proxy?server=185.123.45.67&port=443&secret=ee...
     
     Нажмите на ссылку и выберите "Подключить"
```

### Уведомления

Бот может отправлять автоматические уведомления:

- Сервер стал недоступен
- Сервер снова доступен
- Выполнена ротация секрета
- Добавлен новый сервер

Настройка уведомлений в разделе **Настройки > Telegram-бот > Уведомления**.

## Часто задаваемые вопросы

### Общие вопросы

**Q: Сколько серверов можно добавить?**
A: Ограничений нет. Вы можете добавить столько серверов, сколько у вас есть.

**Q: Можно ли использовать один сервер для нескольких прокси?**
A: Нет, каждый MTProxy работает на отдельном сервере или порту.

**Q: Как узнать, сколько пользователей подключено?**
A: Количество текущих соединений отображается в статистике. Точное число уникальных пользователей недоступно из-за особенностей протокола.

### Про ссылки

**Q: Почему ссылка не работает?**
A: Проверьте:
1. Сервер активен (зеленый статус)
2. Порт 443 открыт на сервере
3. Секрет актуальный (не была ли ротация)
4. Telegram обновлен до последней версии

**Q: Можно ли сделать "красивую" короткую ссылку?**
A: Да, вы можете использовать сервисы сокращения ссылок или настроить редирект на своем домене.

**Q: Ссылка работала, а теперь нет. Что случилось?**
A: Возможные причины:
- Был ротирован секрет
- Сервер недоступен
- IP-адрес сервера заблокирован

### Про ротацию

**Q: Как часто нужно ротировать секреты?**
A: Зависит от ситуации:
- При активных блокировках: каждый день или чаще
- В обычном режиме: раз в неделю или месяц
- При подозрении на утечку: немедленно

**Q: После ротации пользователи отключились. Это нормально?**
A: Да, это ожидаемое поведение. Отправьте им новую ссылку.

### Про безопасность

**Q: Кто может видеть трафик пользователей?**
A: Владелец сервера технически может видеть метаданные (кто подключился, когда), но не содержимое сообщений - оно зашифровано Telegram.

**Q: Безопасно ли использовать публичные прокси?**
A: Для обычного использования Telegram - да. Для конфиденциальной переписки лучше использовать свой сервер.

## Решение проблем

### Сервер показывает статус "Недоступен"

**Проверьте:**

1. **Сервер запущен?**
   - Зайдите на VPS по SSH
   - Выполните: `docker ps` или `systemctl status mtproxy`

2. **Порт открыт?**
   - Проверьте firewall: `ufw status`
   - Проверьте доступность: `nc -zv IP PORT`

3. **IP не заблокирован?**
   - Попробуйте подключиться с другого устройства/сети
   - Проверьте на сайтах проверки блокировок

4. **Правильный IP/порт в панели?**
   - Сравните с реальными настройками сервера

### Ссылка не открывается в Telegram

**Решения:**

1. **Обновите Telegram** до последней версии

2. **Проверьте формат ссылки:**
   - Должна начинаться с `tg://proxy?` или `https://t.me/proxy?`

3. **Попробуйте другой формат:**
   - Если не работает `tg://`, попробуйте `t.me`
   - И наоборот

4. **На iOS:** может потребоваться открыть ссылку через Safari

### Панель управления не загружается

**Проверьте:**

1. **Интернет-соединение**
2. **Правильный адрес панели**
3. **Очистите кэш браузера** (Ctrl+Shift+R)
4. **Попробуйте другой браузер**
5. **Проверьте статус сервера панели**

### Ротация не работает

**Возможные причины:**

1. **Нет связи с сервером**
   - Проверьте SSH доступ к VPS
   - Проверьте API ключи

2. **Ошибка в скриптах**
   - Посмотрите логи ротации
   - Проверьте права доступа

3. **Docker не перезапускается**
   - Зайдите на сервер
   - Выполните: `docker restart mtproxy`

### Бот не отвечает

**Проверьте:**

1. **Токен бота актуален?**
2. **Бот запущен?** - проверьте процесс на сервере
3. **Webhook настроен?** - для режима webhook
4. **Логи бота** - посмотрите ошибки
