## 1. Обзор проекта

### 1.1 Цель проекта

MTProxy Manager — система управления MTProto прокси серверами для Telegram. Система позволяет:

- Централизованно управлять несколькими прокси серверами
- Автоматически ротировать секреты для обхода блокировок
- Генерировать Fake TLS ссылки (маскировка под HTTPS трафик)
- Мониторить статус и логи всех серверов
- Распространять актуальные ссылки через Telegram бота

## 4. Процесс разработки

### 4.1 Подготовка VPS сервера

#### Шаг 1: Подключение к серверу

```bash
# Подключение по SSH
ssh root@YOUR_SERVER_IP

# Или с ключом
ssh -i ~/.ssh/your_key root@YOUR_SERVER_IP
```

#### Шаг 2: Обновление системы

```bash
# Для Ubuntu/Debian
apt update && apt upgrade -y

# Установка необходимых пакетов
apt install -y curl wget git nano htop
```

#### Шаг 3: Установка Docker

```bash
# Установка Docker
curl -fsSL https://get.docker.com | sh

# Добавление пользователя в группу docker
usermod -aG docker $USER

# Установка docker-compose
curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
chmod +x /usr/local/bin/docker-compose

# Проверка установки
docker --version
docker-compose --version
```

#### Шаг 4: Настройка firewall

```bash
# Установка ufw
apt install -y ufw

# Базовые правила
ufw default deny incoming
ufw default allow outgoing

# Разрешить SSH
ufw allow 22/tcp

# Разрешить MTProxy порт
ufw allow 443/tcp

# Включить firewall
ufw enable

# Проверить статус
ufw status
```

### 4.2 Развертывание MTProxy

#### Шаг 1: Создание рабочей директории

```bash
mkdir -p /opt/mtproxy
cd /opt/mtproxy
```

#### Шаг 2: Генерация секрета

```bash
# Генерация случайного секрета (32 hex символа)
SECRET=$(head -c 16 /dev/urandom | xxd -ps)
echo "Сгенерированный секрет: $SECRET"

# Сохранение секрета
echo "$SECRET" > /opt/mtproxy/secret.txt
```

#### Шаг 3: Получение конфигурации Telegram

```bash
# Скачивание конфигурации
curl -s https://core.telegram.org/getProxySecret -o /opt/mtproxy/proxy-secret
curl -s https://core.telegram.org/getProxyConfig -o /opt/mtproxy/proxy-multi.conf
```

#### Шаг 4: Запуск MTProxy контейнера

```bash
# Создание docker-compose.yml
cat > /opt/mtproxy/docker-compose.yml << 'EOF'
version: '3.8'

services:
  mtproxy:
    image: telegrammessenger/proxy:latest
    container_name: mtproxy
    restart: always
    ports:
      - "443:443"
    volumes:
      - ./proxy-secret:/data/proxy-secret
      - ./proxy-multi.conf:/data/proxy-multi.conf
    environment:
      - SECRET=${SECRET}
    command: ["-p", "443", "-S", "${SECRET}", "-M", "1"]
EOF

# Запуск
docker-compose up -d

# Проверка логов
docker logs mtproxy
```

#### Шаг 5: Генерация ссылки подключения

```bash
# Получение внешнего IP
SERVER_IP=$(curl -s ifconfig.me)

# Домен для Fake TLS маскировки
FAKE_TLS_DOMAIN="www.google.com"

# Кодирование домена в hex
DOMAIN_HEX=$(echo -n "$FAKE_TLS_DOMAIN" | xxd -ps)

# Полный секрет с Fake TLS (ee + secret + domain_hex)
FULL_SECRET="ee${SECRET}${DOMAIN_HEX}"

# Генерация ссылки
echo "=== ССЫЛКА ДЛЯ ПОДКЛЮЧЕНИЯ ==="
echo "tg://proxy?server=${SERVER_IP}&port=443&secret=${FULL_SECRET}"
echo ""
echo "=== HTTPS ССЫЛКА ==="
echo "https://t.me/proxy?server=${SERVER_IP}&port=443&secret=${FULL_SECRET}"
```

### 4.4 Настройка автоматизации

#### Шаг 1: Скрипт ротации секрета

```bash
# Создание скрипта на сервере
cat > /opt/mtproxy/rotate-secret.sh << 'EOF'
#!/bin/bash

# Конфигурация
MTPROXY_DIR="/opt/mtproxy"
CONTAINER_NAME="mtproxy"
API_URL="https://your-panel.vercel.app/api"
API_SECRET="your-api-secret"
PROXY_ID="1"

# Генерация нового секрета
NEW_SECRET=$(head -c 16 /dev/urandom | xxd -ps)

# Сохранение
echo "$NEW_SECRET" > "$MTPROXY_DIR/secret.txt"

# Обновление .env файла
echo "SECRET=$NEW_SECRET" > "$MTPROXY_DIR/.env"

# Перезапуск контейнера
cd "$MTPROXY_DIR"
docker-compose down
docker-compose up -d

# Уведомление панели
curl -X POST "$API_URL/proxies/$PROXY_ID/rotate" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $API_SECRET" \
  -d "{\"secret\": \"$NEW_SECRET\"}"

echo "Секрет успешно обновлен: $NEW_SECRET"
EOF

chmod +x /opt/mtproxy/rotate-secret.sh
```

#### Шаг 2: Настройка cron

```bash
# Открыть crontab
crontab -e

# Добавить задачи:

# Ротация секрета каждый день в 3:00
0 3 * * * /opt/mtproxy/rotate-secret.sh >> /var/log/mtproxy-rotate.log 2>&1

# Обновление конфигурации Telegram каждые 6 часов
0 */6 * * * curl -s https://core.telegram.org/getProxyConfig -o /opt/mtproxy/proxy-multi.conf && docker restart mtproxy
```

#### Шаг 3: Настройка Telegram бота

```bash
# Установка Node.js
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install -y nodejs

# Создание директории для бота
mkdir -p /opt/telegram-bot
cd /opt/telegram-bot

# Инициализация проекта
npm init -y
npm install node-telegram-bot-api dotenv

# Создание .env файла
cat > .env << EOF
BOT_TOKEN=your_bot_token_from_botfather
PANEL_API_URL=https://your-panel.vercel.app/api
API_SECRET=your-api-secret
EOF

# Копирование скрипта бота
cp /path/to/telegram-bot.js ./bot.js

# Создание systemd сервиса
cat > /etc/systemd/system/telegram-bot.service << EOF
[Unit]
Description=MTProxy Telegram Bot
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/telegram-bot
ExecStart=/usr/bin/node bot.js
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
EOF

# Запуск бота
systemctl daemon-reload
systemctl enable telegram-bot
systemctl start telegram-bot

# Проверка статуса
systemctl status telegram-bot
```

### 6.2 Решение типичных проблем

#### Проблема: Прокси не подключается

```bash
# 1. Проверить, запущен ли контейнер
docker ps | grep mtproxy

# 2. Проверить логи контейнера
docker logs mtproxy --tail 50

# 3. Проверить, открыт ли порт
netstat -tlnp | grep 443

# 4. Проверить firewall
ufw status

# 5. Перезапустить контейнер
docker restart mtproxy
```

#### Проблема: Бот не отвечает

```bash
# 1. Проверить статус сервиса
systemctl status telegram-bot

# 2. Проверить логи
journalctl -u telegram-bot --tail 50

# 3. Проверить токен бота
cat /opt/telegram-bot/.env

# 4. Перезапустить бота
systemctl restart telegram-bot
```

#### Проблема: Панель недоступна

1. Проверьте статус деплоя в Vercel Dashboard
2. Проверьте логи функций в Vercel
3. Проверьте переменные окружения
4. Попробуйте редеплой

### 6.3 Мониторинг

#### Простой мониторинг через cron

```bash
# Добавить в crontab
*/5 * * * * /opt/mtproxy/check-health.sh
```

```bash
# /opt/mtproxy/check-health.sh
#!/bin/bash

BOT_TOKEN="your_bot_token"
ADMIN_CHAT_ID="your_telegram_id"

# Проверка контейнера
if ! docker ps | grep -q mtproxy; then
    curl -s -X POST "https://api.telegram.org/bot$BOT_TOKEN/sendMessage" \
        -d "chat_id=$ADMIN_CHAT_ID" \
        -d "text=⚠️ MTProxy контейнер не запущен! Перезапускаю..."
    
    cd /opt/mtproxy && docker-compose up -d
fi
```
