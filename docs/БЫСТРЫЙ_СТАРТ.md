# Быстрый старт: MTProxy Manager

## Требования

- VPS сервер (Ubuntu 20.04+), 1 vCPU, 1GB RAM
- Открытый порт 443
- Vercel аккаунт
- Telegram бот токен от @BotFather

---

## Шаг 1: Установка на сервер (5 минут)

```bash
# Подключиться к серверу
ssh root@YOUR_IP

# Скачать и запустить установщик
curl -sSL https://raw.githubusercontent.com/YOUR_REPO/main/server-scripts/setup-mtproxy.sh | bash
```

После завершения скрипт выведет:
- Секрет прокси
- Ссылку для подключения
- Инструкции по дальнейшей настройке

---

## Шаг 2: Деплой панели на Vercel (3 минуты)

1. Форкните репозиторий на GitHub
2. Зайдите на [vercel.com/new](https://vercel.com/new)
3. Импортируйте форк
4. Добавьте переменные окружения:
   ```
   API_SECRET=ваш_случайный_ключ
   ADMIN_PASSWORD=ваш_пароль
   ```
5. Нажмите Deploy

---

## Шаг 3: Добавление прокси в панель (2 минуты)

1. Откройте панель по URL от Vercel
2. Нажмите "Добавить прокси"
3. Введите данные с сервера:
   - IP адрес сервера
   - Порт: 443
   - Секрет (из вывода установщика)
   - Fake TLS домен: www.google.com

---

## Шаг 4: Настройка Telegram бота (5 минут)

На сервере:

```bash
# Перейти в директорию бота
cd /opt/telegram-bot

# Отредактировать настройки
nano .env
```

Заполнить:
```
BOT_TOKEN=токен_от_botfather
PANEL_API_URL=https://ваша-панель.vercel.app/api
API_SECRET=тот_же_ключ_что_в_vercel
```

Запустить:
```bash
systemctl start telegram-bot
```

---

## Готово!

### Проверка работы

1. **Прокси**: Добавьте ссылку из панели в Telegram → Настройки → Прокси
2. **Бот**: Напишите боту `/proxy` для получения ссылок
3. **Панель**: Проверьте статус в веб-интерфейсе

### Полезные команды

```bash
# Статус прокси
docker ps

# Логи прокси
docker logs mtproxy -f

# Ручная ротация секрета
/opt/mtproxy/rotate-secret.sh

# Перезапуск бота
systemctl restart telegram-bot
```
