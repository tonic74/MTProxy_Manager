# Пошаговое руководство по деплою

## Часть 1: Vercel (веб-панель)

### 1. Регистрация на Vercel

1. Откройте https://vercel.com
2. Нажмите "Sign Up"
3. Выберите "Continue with GitHub"
4. Авторизуйте Vercel

### 2. Импорт проекта

1. На главной странице Vercel нажмите **"Add New Project"**
2. В списке репозиториев найдите **tonic74/MTProxy_Manager**
3. Нажмите **"Import"**

### 3. Настройка переменных окружения

В разделе **"Environment Variables"** добавьте 3 переменные:

#### Переменная 1: API_SECRET
- **Name**: `API_SECRET`
- **Value**: `AcKx5ZzCotRghOYnl7p1W2DwEFyPrU8T`

#### Переменная 2: ADMIN_PASSWORD
- **Name**: `ADMIN_PASSWORD`
- **Value**: `<придумайте надежный пароль>`
- Пример: `MySecurePass2026!`

#### Переменная 3: DATABASE_URL
- **Name**: `DATABASE_URL`
- **Value**: `postgresql://user:password@host:port/database`

Для получения DATABASE_URL:
1. Создайте проект на https://neon.tech
2. Создайте базу данных `mtproxy`
3. Скопируйте connection string из панели Neon

### 4. Деплой

1. Нажмите **"Deploy"**
2. Дождитесь завершения (2-3 минуты)
3. После успешного деплоя скопируйте URL проекта
   - Пример: `https://mtproxy-manager-abc123.vercel.app`

### 5. Сохраните данные

```
URL панели: _______________________________
API_SECRET: AcKx5ZzCotRghOYnl7p1W2DwEFyPrU8T
ADMIN_PASSWORD: _______________________________
```

## Часть 2: Сервер (MTProxy + Telegram бот)

### Подготовка

Вам понадобится:
- IP адрес сервера
- SSH ключ (уже есть в `.ssh/123`)
- URL панели Vercel (из Части 1)
- Токен бота: `8780642333:AAHxevHHibY7ufkRfMQ4r9Dmx14qphyqnXE`
- Ваш Telegram ID (получите через @userinfobot)

### 1. Получите ваш Telegram ID

1. Откройте Telegram
2. Найдите бота **@userinfobot**
3. Отправьте `/start`
4. Скопируйте ваш ID (например: `123456789`)

### 2. Отредактируйте скрипт установки

Откройте файл `SERVER_SETUP.sh` и заполните:

```bash
PANEL_URL="https://ваша-панель.vercel.app"  # URL из Части 1
ADMIN_TELEGRAM_ID="ваш_telegram_id"         # ID из шага 1
```

### 3. Подключитесь к серверу

```bash
# Windows PowerShell
ssh -i .ssh/123 root@IP_СЕРВЕРА
```

Замените `IP_СЕРВЕРА` на реальный IP.

### 4. Загрузите и запустите скрипт

На сервере выполните:

```bash
# Скачать скрипт
curl -sSL https://raw.githubusercontent.com/tonic74/MTProxy_Manager/main/SERVER_SETUP.sh -o setup.sh

# Отредактировать (добавить PANEL_URL и ADMIN_TELEGRAM_ID)
nano setup.sh

# Запустить
chmod +x setup.sh
sudo bash setup.sh
```

### 5. Сохраните данные прокси

После установки скрипт выведет:

```
IP: X.X.X.X
Port: 443
Secret: ddXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX
TG Link: tg://proxy?server=...
Web Link: https://t.me/proxy?server=...
```

Скопируйте эти данные - они понадобятся для добавления в панель.

## Часть 3: Добавление прокси в панель

### 1. Откройте веб-панель

Перейдите по URL из Части 1.

### 2. Добавьте прокси

1. Нажмите кнопку **"Добавить прокси"**
2. Заполните форму:
   - **Название**: `Germany-1`
   - **IP адрес**: IP вашего сервера
   - **Порт**: `443`
   - **Секрет**: секрет из вывода скрипта (начинается с `dd`)
   - **Fake TLS домен**: `google.com`
3. Нажмите **"Добавить"**

### 3. Скопируйте ссылку

После добавления прокси появится в таблице. Нажмите на кнопку копирования ссылки.

## Часть 4: Проверка работы

### 1. Проверьте прокси в Telegram

1. Откройте Telegram на телефоне
2. Перейдите: **Настройки → Данные и память → Тип соединения**
3. Нажмите **"Использовать прокси"**
4. Нажмите **"Добавить прокси"**
5. Вставьте ссылку из панели
6. Убедитесь, что статус **"Подключено"**

### 2. Проверьте Telegram бота

1. Найдите вашего бота в Telegram (имя бота вы создали через @BotFather)
2. Отправьте `/start`
3. Отправьте `/proxies` - должен вернуть список прокси
4. Отправьте `/stats` - должен показать статистику

### 3. Проверьте веб-панель

1. Откройте панель
2. Проверьте, что прокси отображается со статусом **"Активен"**
3. Попробуйте нажать **"Ротация всех"** - секрет должен обновиться

## Решение проблем

### Прокси не подключается

```bash
# На сервере проверьте:
docker ps | grep mtproxy          # Контейнер запущен?
docker logs mtproxy --tail 50    # Есть ошибки?
ufw status                        # Порт 443 открыт?
```

### Бот не отвечает

```bash
# На сервере проверьте:
systemctl status telegram-bot     # Сервис запущен?
journalctl -u telegram-bot -n 50  # Есть ошибки?
cat /opt/telegram-bot/.env        # Токен правильный?
```

### Панель не открывается

1. Проверьте статус деплоя в Vercel Dashboard
2. Проверьте логи функций (Functions → Logs)
3. Убедитесь, что все переменные окружения добавлены
